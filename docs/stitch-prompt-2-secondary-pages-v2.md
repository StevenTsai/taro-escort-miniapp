# 壹鹿康行 - Stitch UI 设计提示词 V2（第二轮：扩展页面）

## 前置约束

请严格继承第一轮《设计系统 + 核心页面》的全部视觉规范、组件样式、间距体系、色彩体系和页面气质。

这一轮不是重新定义风格，而是在同一套设计系统下继续扩展页面。请重点保证：

- 风格不漂移
- 卡片不变形
- 表单语言一致
- 按钮、Tag、浮动按钮、弹窗样式完全延续第一轮
- 所有页面仍然是 375 x 812 的微信小程序高保真页面

不要新增本项目中不存在的业务流程、导航入口或内容模块。

---

## 本轮目标

请继续设计剩余业务页面，使整个小程序形成可连续出图、可直接交付前端还原的完整页面集。

页面分为两类：
- P1：高频核心业务页面
- P2：补足流程闭环的支撑页面

---

## 页面复用与一致性要求

本轮页面必须复用第一轮已有设计语言：

- 陪诊师卡片样式继承自“陪诊师预约列表”
- 订单卡片样式继承自“订单列表”
- 所有表单页继承统一表单卡片样式
- 所有新增按钮、删除按钮、底部固定栏风格统一
- 患者模块所有页面保持为同一信息架构体系
- 报告页虽然是特殊排版，但视觉基因仍然属于同一个产品

如果遇到新页面，请优先复用现有组件，而不是创造新的视觉模式。

---

## 示例数据要求

请沿用第一轮的中文示例数据风格，并补充：

## 第二轮需要输出的页面

### 1. 陪诊师详情页

页面来源：从陪诊师预约列表进入。

内容包括：
- 顶部陪诊师资料卡片：头像、姓名、擅长标签、个人简介
- 可预约服务区域：服务名、服务描述、价格、预约按钮
- 用户评价区域：用户名、星级评分、评论内容、评论时间
- 无评价状态

要求：
- 与陪诊师列表页视觉衔接自然
- 页面要让用户快速建立信任感

### 2. 服务详情页

内容包括：
- 顶部服务封面图
- 服务名称 + 价格
- 3 个 Tab：服务内容 / 服务流程 / 退款说明

其中：
- 服务内容：富文本信息 + 服务项目列表
- 服务流程：竖向步骤条，分诊前 / 诊中 / 诊后
- 退款说明：清晰可读，层级明确

底部固定栏：
- 左侧价格
- 右侧“立即预约”按钮

要求：
- 这是强转化页面，重点突出价格、内容和预约 CTA

### 3. 医院详情页

内容包括：
- 顶部医院封面图，底部渐变遮罩
- 医院名称 + 等级 Tag
- 地址与电话信息
- 特色科室横滑标签
- 医院介绍，支持展开/收起
- 可预约服务列表

要求：
- 医院信息可信赖、结构清晰
- 不要做成资讯文章页

### 4. 预约下单页

这是关键转化表单页。

内容包括：
- 已选服务信息卡片
- 联系信息表单区：
- 就诊人姓名
- 联系方式
- 陪诊城市
- 服务日期时间
- 服务要求
- 服务信息表单区：
- 就诊医院
- 陪诊师
- 提示文案
- 协议勾选区
- 底部固定价格栏 + 立即支付按钮

需要体现：
- 默认填写态
- loading 支付按钮
- 微信支付流程示意

要求：
- 表单层级要轻松、可信、不会让用户紧张

### 5. 患者管理列表

内容包括：
- 患者卡片：姓名、性别、年龄、联系方式
- 左滑删除状态
- 分页加载感
- 右下角浮动新增按钮
- 空状态

要求：
- 清晰体现“管理列表”属性
- 删除交互要有安全感，不要显得危险突兀

### 6. 陪诊师订单列表

这是陪诊师视角的“我的陪诊”。

内容包括：
- 顶部状态 Tab：已接单 / 进行中 / 已取消 / 已完成
- 复用订单卡片样式
- 空状态

要求：
- 让人一眼看出和用户端订单列表是同一体系，但角色视角不同

### 7. 患者详情页

内容包括：
- 顶部患者信息卡片
- 基础病史区
- 最近治疗事件区
- 查看全部入口
- 右下角浮动新增按钮
- ActionSheet：添加治疗小结 / 检查事件 / 用药事件 / 通用事件

要求：
- 这是患者管理的中枢页面
- 信息多，但不要压迫

### 8. 事件详情页

请输出 4 种事件类型的详情表现：
- 治疗小结
- 检查事件
- 用药事件
- 通用事件

内容包括：
- 顶部事件类型 Tag + 标题 + 日期
- 对应类型的详细内容
- 图片附件预览
- 底部操作区：编辑 / 删除

要求：
- 4 种类型虽然内容不同，但页面骨架尽量统一

### 9. 病程时间线页

内容包括：
- 顶部筛选栏：日期范围 + 事件类型
- 竖向时间轴
- 事件卡片列表
- 上拉加载更多
- 下拉刷新
- 右下角浮动新增按钮

要求：
- 时间线要有医疗记录感，但仍然清爽

### 10. 问诊病历报告页

这是一个特殊页面，但仍属于同一产品。

内容包括：
- 头部：品牌名、二维码、报告标题
- 患者体格信息
- 内容区块：
- 疾病诊断
- 治疗总结
- 基础病史
- 医疗事件
- 检查报告
- 底部固定“下载 PDF”按钮
- 加载态
- 失败重试态

要求：
- 类似纸质报告排版，但不能脱离整个产品气质
- 信息可读性优先

### 11. 陪诊师收入管理页

内容包括：
- 顶部概览卡片：本月收入、总收入、订单数
- 筛选栏：月份选择、状态筛选
- 收入明细列表
- 空状态

要求：
- 数据感明确
- 但仍然保持温和而不是财务后台风

### 12. 陪诊师注册页

3 步流程页：

#### 第 1 步：基本信息
- 头像上传
- 姓名
- 性别
- 城市
- 手机号

#### 第 2 步：身份认证
- 身份证正面
- 身份证反面
- 上传中状态

#### 第 3 步：服务信息
- 服务标签多选
- 服务介绍

底部按钮：
- 上一步
- 下一步 / 提交审核

要求：
- 明确、友好、降低填写焦虑

### 13. 陪诊师资料编辑页

内容包括：
- 头像上传区
- 昵称输入
- 服务标签多选
- 服务介绍输入
- 底部固定保存按钮

要求：
- 风格与注册页高度一致，但更轻量

### 14. 创建 / 编辑患者页

同一页面的两种模式。

内容包括：
- 姓名
- 性别
- 年龄
- 身高
- 体重
- 联系电话
- 既往病史
- 过敏史
- 家族病史
- 疾病名称
- 治疗总结
- 底部保存 / 更新按钮

要求：
- 表单很长，但必须清楚且可继续填写

### 15. 添加事件页面

请输出以下 4 个页面，并保持统一框架：
- 添加治疗小结
- 添加检查事件
- 添加用药事件
- 添加通用事件

统一要求：
- 白色表单卡片
- 顶部字段清晰
- 图片附件上传区域统一
- 底部保存按钮统一
- 编辑模式可复用同一页面结构

各页面特有内容：

#### 添加治疗小结
- 事件日期
- 标题
- 医院名称
- 医生姓名
- 诊断结果
- 治疗方案
- 是否关键节点
- 图片附件

#### 添加检查事件
- 事件日期
- 标题
- 检查机构
- 检查项目
- 检查结果
- 备注
- 图片附件

#### 添加用药事件
- 事件日期
- 标题
- 药品列表动态增删
- 每条药品包括：名称、剂量、用法、起止日期
- 图片附件

#### 添加通用事件
- 事件日期
- 事件标题
- 事件描述
- 图片附件

## 本轮状态页与关键交互要求

请在设计中适当体现以下关键状态，不需要每页都单独展开成很多画板，但关键页要体现：

- 图片上传中
- 表单校验失败
- 左滑删除露出按钮
- 报告加载中
- 报告生成失败
- 图片点击后可全屏预览
- 步骤条切换
- Tab 切换

---

## 微交互要求

- 步骤条切换：当前步骤轻微放大，连接线渐进填充
- 图片上传：点击后调起 ActionSheet，上传中显示进度或 loading
- 左滑删除：红色删除区域清晰，但不夸张
- Tab 切换：指示条平滑滑动
- 药品条目增删：新增向下展开，删除平滑收起
- 表单报错：输入框红边 + 底部错误提示

仍然遵循原则：动画服务于理解，不是表演。

---

## 输出格式要求

- 所有页面为高保真中文小程序页面
- 所有页面保持与第一轮完全同源的设计语言
- 关键流程页尽量体现真实数据
- 不要输出代码
- 不要输出项目中不存在的新页面
- 不要加入课程、AI、优惠券、收藏等当前路由中不存在的页面
