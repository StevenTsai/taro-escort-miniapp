# 壹鹿康行 — Stitch UI 设计（第二轮：扩展页面）

> 前置条件：请基于第一轮的设计系统和核心页面风格，保持视觉一致性，继续设计以下页面。
> 主色 #008000，页面背景 #F5F6FA，卡片白底圆角 16rpx + 柔和阴影，调性：专业、温暖、可信赖。

## P1 重要页面

### 页面 6.5：陪诊师详情页

> 从陪诊师预约列表点击"立即预约"或点击陪诊师卡片进入

1. **顶部陪诊师资料区**（白色卡片）：
   - 居中圆形头像 80rpx
   - 姓名（H2 加粗居中）
   - 擅长标签列表（Tag 横排，绿色圆角 Tag，超出换行）
   - 个人简介（正文灰色，最多 3 行，可展开）
2. **可预约服务区域**：
   - 标题行："预约服务" + 右侧绿色圆点装饰
   - 服务卡片列表（每个卡片：左侧服务名称 + 服务描述，右侧价格（绿色加粗）+ "预约"绿色小按钮）
   - 点击"预约"按钮跳转预约下单页，并预选该陪诊师 ID
3. **用户评价区域**：
   - 标题行："用户评价" + 右侧绿色圆点装饰
   - 评价卡片列表（每条：左侧用户名 + 右侧星级评分（只读，金色星星）+ 下方评论内容文字 + 底部评论时间灰色小字）
   - 无评价时显示"暂无评价"（居中灰色文字）

### 页面 7：服务详情

1. 顶部服务封面图（全宽，宽高比 2:1）
2. 服务名称（页面标题 36rpx 加粗）+ 价格（绿色 32rpx 加粗，带 ¥ 前缀）
3. Tab 切换：服务内容 / 服务流程 / 退款说明，选中态绿色文字 + 底部指示条
4. **服务内容 Tab**：富文本段落，包含服务描述（如"半天陪诊：专业陪诊师陪同就诊服务4小时"）和具体服务项目列表
5. **服务流程 Tab**：竖向步骤条，每个节点：圆形序号（绿色背景白字）+ 竖向连接线 + 标题 + 描述文字。流程分三阶段（诊前→诊中→诊后），不同服务类型流程不同：
   - 陪诊服务：诊前（1对1服务、协助挂号、行程规划）→ 诊中（全程陪同、问诊辅助、实时反馈、情绪安抚）→ 诊后（专病解读）
   - 代问诊：诊前（诊前咨询、资料准备）→ 诊中（代问诊）→ 诊后（诊后关怀）
   - 病理会诊：诊前（会诊准备）→ 诊中（提交病理资料、取会诊结果、代问诊）→ 诊后（健康管理）
   - 代办服务：诊前（1对1服务、协助挂号）→ 诊中（实时反馈）→ 诊后（健康管理）
6. **退款说明 Tab**：富文本段落
7. 底部固定栏：左侧价格展示 + 右侧"立即预约"绿色主按钮

### 页面 8：医院详情

1. 顶部医院封面图（全宽，底部渐变遮罩，医院名叠在遮罩上白色大字）
2. 医院名称（H1）+ 等级 Tag（如"三甲"绿色 Tag）
3. 信息行：📍 地址（绿色可点击文字，跳转地图导航）| 📞 电话（绿色可点击文字，拨打电话）
4. 特色科室：横滑 Tag 列表（浅绿底色）
5. 医院介绍：默认显示 3 行，点击"展开"显示全文，收起时末尾渐变遮罩
6. 可预约服务列表：标题"可预约服务"。卡片式，左侧服务名 + 描述，右侧价格（绿色）+ 预约按钮（绿色小按钮）。每个服务卡片点击跳转服务详情页

### 页面 9：预约下单

1. **已选服务信息卡片**（顶部白色卡片）：左侧服务封面缩略图（圆角 10rpx）+ 右侧服务名称 + 价格（绿色加粗），加载中时显示"加载中..."和"¥--"
2. **联系信息表单区**（白色卡片，标题"联系方式"）：
   - 就诊人姓名（输入框，必填，placeholder "请输入就诊人姓名"）
   - 联系方式（输入框，必填，手机号验证：1 开头 11 位数字，placeholder "请输入联系方式"）
   - 陪诊城市（城市选择器组件，弹出省市区三级联动，必填）
   - 服务日期时间（日期时间选择器组件，分钟只能选 00 或 30（30 分钟步长），起始时间为当前时间向上取整到最近的 00 或 30 分，必填）
   - 服务要求（多行文本框，选填，placeholder "请输入服务要求"）
3. **医院和陪诊师表单区**（白色卡片，标题"服务信息"）：
   - 就诊医院（医院选择器组件，根据已选城市筛选，必填）
   - 陪诊师（陪诊师选择器组件，根据已选城市筛选，选填，可不选由系统分配）
4. **提示文案**："支付后您的专属陪诊助理会在第一时间联系您"（居中灰色小字）
5. **协议勾选区**：Checkbox + "我已阅读并同意《用户服务协议》和《隐私政策》"（协议名绿色可点击文字，点击下载并打开对应文档）
6. **底部固定栏**：左侧价格展示（¥ + 绿色大字）+ 右侧"立即支付"绿色主按钮（loading 态显示旋转图标）
7. **支付流程**：点击"立即支付" → 创建订单 → 调起微信支付 → 支付成功 Toast / 支付失败 Toast / 用户取消无操作

### 页面 10：患者管理列表

1. **患者卡片**：白色卡片，左侧 4rpx 绿色竖条装饰，姓名（加粗）+ 性别 Tag（男蓝 #1890FF / 女粉 #FF69B4）+ 年龄，下方联系方式（灰色小字），右侧箭头

5. **右下角浮动"+"按钮**：绿色圆形，白字"+"，阴影 float
6. **空状态**：医疗主题插画 + "还没有患者信息" + "添加第一位患者"绿色按钮
