# 壹鹿康行 - Stitch UI 设计提示词 V2（第一轮：设计系统 + 核心页面）

## 使用目标

请为微信小程序“壹鹿康行”设计一组可连续出图、风格统一、可直接用于前端还原的高保真移动端页面。

这是一个医疗陪诊服务小程序，面向两类用户：
- 普通用户：预约陪诊服务、管理患者、查看订单
- 陪诊师：接单服务、管理收入、编辑资料

请严格基于以下要求输出，不要新增当前项目中不存在的业务模块、导航入口或营销板块。

---

## 输出总原则

- 设备场景：微信小程序移动端
- 画板尺寸：375 x 812
- 语言：简体中文
- 模式：浅色模式
- 设计目标：专业、温暖、可信赖、易读、清晰，不要炫技，不要夸张视觉噪音
- 输出风格：高保真 UI，而不是线框图
- 输出一致性：所有页面必须像同一个设计系统下的同一产品
- 优先级：先保证信息层级和一致性，再追求装饰感

---

## 品牌与视觉方向

- 品牌名：壹鹿康行
- 行业：医疗陪诊服务
- 产品关键词：就医协助、流程清晰、安心陪伴、可信赖
- 视觉参考方向：医疗健康产品的专业感 + 服务型平台的流程感 + 家庭陪伴式温度感
- 页面气质：现代简洁、轻医疗感、卡片式布局、留白舒适、强调可操作性

请避免：
- 互联网金融风的强对比和重质感
- 过度拟物、玻璃拟态、复杂渐变堆叠
- 潮流电商风、游戏化风格
- 与医疗场景不符的高饱和撞色

## 统一视觉约束

### 色彩

- 主色：#008000
- 主色浅底：#E8F5E9
- 主色按下态：#006400
- 页面背景：#F5F6FA
- 卡片背景：#FFFFFF
- 主标题文字：#1A1A1A
- 正文文字：#666666
- 辅助文字：#999999
- 分割线：#EEEEEE
- 高亮提示底色：#F0F7F0
- 警示色：#FF4D4F
- 成功色：#52C41A
- 信息色：#1890FF
- 待确认状态：#FAAD14
- 评分星级：#F89D00

### 字体层级

- 页面主标题：36rpx / 700
- 模块标题：32rpx / 600
- 列表标题：28rpx / 600
- 正文：28rpx / 400
- 辅助信息：24rpx / 400
- 状态与时间：22rpx / 400
- 金额数字可以更强调，但不要脱离整体体系

### 间距与圆角

- 页面左右边距：24rpx
- 卡片间距：20rpx
- 卡片内边距：24rpx
- 卡片圆角：16rpx
- 大卡片圆角：24rpx
- 标签与按钮：药丸圆角
- 图片小圆角：10rpx 到 12rpx

### 阴影

- 常规卡片：0 4rpx 16rpx rgba(0,0,0,0.06)
- 浮层 / 浮动按钮 / 弹窗：0 8rpx 24rpx rgba(0,0,0,0.12)

### 图标与插画

- 图标风格：统一的线性医疗图标，圆角友好，简洁，不要拟物
- 插画风格：浅色扁平医疗插画，用于空状态
- Banner 和封面图风格：真实医疗服务场景照片感或半写实插图，但整体要干净、温和
- 不要使用 3D 图标、赛博风图形、夸张霓虹效果

## 全局组件规则

所有页面必须复用同一套组件语言。以下组件在所有页面中必须保持一致，不允许每页重新发明一套样式：

- 底部 TabBar
- 顶部搜索栏
- 主按钮 / 次按钮 / 危险按钮
- 状态 Tag
- 列表卡片
- 表单卡片
- 空状态
- Loading 状态
- Dialog
- Toast
- ActionSheet

### 底部 TabBar

- 4 个 Tab：首页 / 陪诊师预约 / 患者管理 / 我的
- 高度 100rpx，适配底部安全区
- 顶部 1rpx 分割线
- 选中态：绿色图标 + 绿色文字
- 未选中态：灰色图标 + 灰色文字
- 风格简洁，不做夸张浮起效果

### 按钮

- 主按钮：绿色实底，白字，高度 88rpx
- 次按钮：白底、绿色描边、绿色字
- 危险按钮：红底白字
- 禁用态：40% 透明度
- 点击反馈：缩放 0.96，150ms

### Tag

- 药丸形
- 文字尺寸 22rpx
- 状态类 Tag 使用对应状态色
- 普通标签使用主色浅底 + 主色文字

### 卡片

- 白底
- 圆角 16rpx
- 柔和阴影
- 内边距统一
- 标题、正文、辅助信息层级稳定

### 空状态

- 医疗主题浅色插画
- 主文案 + 副文案 + 可选 CTA 按钮
- 不要只放一个图标和一句话

### Loading

- 列表：骨架屏
- 按钮：按钮内旋转 loading
- 全页：中性、不刺眼的加载态

### Dialog

- 标题居中
- 内容可读性强
- 双按钮布局：取消 / 确认
- 适合退款规则、退出登录、删除确认

### Toast

- 黑色半透明圆角气泡
- 图标 + 文字
- 2 秒自动消失

### ActionSheet

- 底部弹出
- 顶部圆角
- 每行等高
- 取消按钮独立分组

## 页面复用规则

请在页面设计中显式保持这些复用关系：

- 首页医院卡片与医院详情页中的服务卡片风格一致
- 订单列表与陪诊师订单列表复用同一类订单卡片语言
- 所有表单页复用同一类表单卡片、输入框、标题样式
- 患者相关页面共用同一组信息卡片、事件卡片、浮动新增按钮
- 个人中心、陪诊师编辑、陪诊师注册中的头像样式保持一致

## 示例数据风格

为避免输出过于空泛，请在设计稿中使用接近真实产品的中文示例数据：

- 纯 lorem ipsum
- 无意义编号堆砌

## 第一轮需要输出的页面

请按顺序输出以下页面，每个页面都要是高保真移动端小程序页面。

### 1. 设计系统预览页

内容包括：
- 色板
- 字体层级
- 按钮
- Tag
- 卡片
- Dialog
- Toast
- ActionSheet
- 空状态
- Loading 状态

目标：
- 作为后续所有页面的母版规范页

### 2. TabBar 组件规范

内容包括：
- 4 个 Tab 的默认态和选中态
- 图标与文字关系
- 底部安全区表现

### 3. 首页

从上到下布局：

1. 顶部：左侧城市名 + 下拉箭头，右侧搜索图标
2. 搜索栏：placeholder 为“搜索医院、服务”
3. Banner 轮播：2:1 比例，圆角 16rpx
4. 引导文案：左侧“还不清楚什么是陪诊？”；右侧“点击查看陪诊介绍 >”
5. 服务入口：
- 上排 2 个大入口：半天陪诊、代问诊
- 下排 3 个小入口：全天陪诊、病理会诊、代办服务
- 所有入口点击跳转服务详情页
6. 合作医院列表：
- 标题“附近医院” + “查看全部”
- 医院卡片内容：封面图、名称、等级标签、地址、特色科室标签

要求：
- 首页要体现“医疗服务平台”的可信赖感
- 不要做成电商首页

### 4. 个人中心

请输出 3 个状态，保持同一页面骨架：

#### 未登录态

- 顶部绿色渐变背景
- 默认头像
- 授权手机号登录按钮

#### 普通用户已登录态

- 头像、昵称、手机号
- 菜单项：
- 我的订单
- 使用说明
- 订阅服务进度
- 退出登录
- 底部“成为陪诊师”浮动卡片
- 右下角客服按钮

#### 陪诊师已登录态

- 菜单项：
- 我的陪诊
- 我的收入
- 编辑资料
- 使用说明
- 订阅服务进度
- 退出登录
- 不显示“成为陪诊师”卡片

### 5. 陪诊师预约列表

内容包括：
- 顶部标题“陪诊师预约”
- 城市筛选标签，可横滑
- 陪诊师卡片：头像、姓名、评分、服务次数、擅长标签、立即预约按钮
- 右下角“成为陪诊师”浮动按钮

要求：
- 卡片风格要适合后续无缝衔接到陪诊师详情页

### 6. 订单列表

内容包括：
- 顶部状态 Tab：全部 / 待确认 / 已支付 / 已接单 / 进行中 / 已取消 / 已完成
- 订单卡片：
- 订单号
- 状态 Tag
- 服务类型
- 陪诊师信息
- 服务时间
- 金额
- 底部操作按钮

需要体现的状态：
- 待确认
- 已支付
- 已接单
- 进行中
- 已完成
- 已取消

额外输出：
- 退款规则 Dialog
- 空状态

### 7. 评论页

内容包括：
- 顶部订单信息卡片
- 星级评分区域
- 评分描述文字
- 评论输入框
- 字数统计
- 提交按钮

要求：
- 评分组件要有明确的可点选感

### 8. 服务介绍页

这是营销落地页，但仍要保持医疗产品气质。

内容包括：
- 顶部 Banner
- “壹鹿康行能解决哪些问题”
- “陪诊是什么”
- 2 列问题场景网格
- “为什么选壹鹿康行”
- 品牌优势描述
- 优势列表
- 有 / 无陪诊对比
- 三步预约引导
- 底部“返回选择服务”按钮

要求：
- 信息很多，但排版不要像长文档，要有节奏
- 不要做成纯运营海报风

### 9. 状态规范页

统一输出：
- 空状态
- 列表加载态
- 按钮 loading 态
- 错误态

该页作为后续页面的状态参考。

## 微交互要求

- 按钮点击：缩放 0.96
- 卡片点击：轻微缩放 0.98
- 页面切换：右向左滑入
- 列表加载：骨架屏 pulse
- Toast：顶部滑入或淡入
- Dialog：居中缩放弹入
- ActionSheet：底部上滑出现
- 星级评分：点击时轻微弹跳

微交互是辅助，不要让动画喧宾夺主。

---

## 输出格式要求

请严格按以下方式理解并输出：

- 所有页面必须是同一套视觉体系
- 所有页面都按小程序页面完整出图，而不是零散组件拼贴
- 关键页面请体现真实示例数据
- 必要时同一页面可在一个画板中展示 2 到 3 个关键状态，但不要杂乱
- 不要新增当前项目未定义页面
- 不要输出代码，只输出设计页面
