# 视频录播功能数据表设计

## 1. 概述

本设计基于视频录播功能需求文档，定义了支持医疗陪护小程序视频录播功能所需的数据表结构。

| 字段名 | 数据类型 | 约束 | 描述 |

| id | VARCHAR(36) | PRIMARY KEY | 分类ID |
| name | VARCHAR(255) | NOT NULL | 分类名称 |

| course_count | INT | DEFAULT 0 | 该分类下的课程数量 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

## 4. 业务规则

1. **课程类型**：
   - `free`：所有用户均可观看
   - `paid`：仅购买过该课程的用户可观看
   - `member`：仅会员用户可观看

2. **视频权限**：
   - 视频可以独立设置是否免费（`is_free`）
   - 即使是付费课程，也可以包含免费视频

3. **课程状态**：
   - `published`：已发布，用户可访问
   - `draft`：草稿，仅管理员可访问

4. **观看历史**：
   - 记录用户观看视频的进度
   - 支持断点续播功能
   - 按观看时间倒序排列

5. **收藏功能**：
   - 用户可以收藏/取消收藏课程
   - 收藏状态通过`user_favorites`表维护

6. **购买功能**：
   - 用户购买课程后，通过`user_courses`表记录
   - 购买后永久有效

## 6. 注意事项

1. 所有表都使用了`created_at`和`updated_at`字段，用于记录数据的创建和更新时间
2. 外键约束确保了数据的完整性和一致性
3. 适当的索引可以提高查询性能
4. 敏感数据（如用户ID）应进行适当的加密或脱敏处理
5. 视频URL和图片URL应考虑使用CDN加速
6. 定期清理过期的观看历史数据，以优化数据库性能

## 7. 扩展建议

3. 考虑添加`course_tags`表，支持课程标签功能
