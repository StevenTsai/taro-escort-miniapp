# 极简版代金券功能接口文档

## 1. 文档概述

### 1.1 文档目的
本文档定义了极简版代金券功能的API接口规范，包括接口列表、请求响应格式、参数说明等，为前后端开发提供统一的接口标准。

### 1.2 接口范围
- 仅包含代金券相关的查询接口
- 支持按状态获取代金券列表
- 支持获取代金券详情
- 无任何修改操作接口

### 1.3 术语定义
| 术语 | 定义 |
|------|------|
| 代金券 | 用户已拥有的优惠券，可按状态分类查看 |
| 已领取 | 代金券已获得且在有效期内，未使用 |
| 已使用 | 代金券已被使用 |
| 已过期 | 代金券超出有效期 |

## 2. 接口设计原则

### 2.1 设计风格
- RESTful API设计风格
- 统一的错误处理
- 清晰的参数命名
- 完整的响应格式

### 2.2 基础URL
- 测试环境：`http://test-api.example.com`
- 生产环境：`https://api.example.com`

### 2.3 请求头
| 字段 | 类型 | 必选 | 说明 |
|------|------|------|------|
| Content-Type | string | 是 | 固定值：`application/json` |
| Authorization | string | 是 | 用户登录凭证，格式：`Bearer {token}` |
| x-biz | string | 是 | 业务标识，固定值：`medical-chaperon` |

### 2.4 响应格式

#### 2.4.1 成功响应
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

#### 2.4.2 失败响应
```json
{
  "code": 400,
  "message": "错误信息",
  "data": null
}
```

#### 2.4.3 错误码说明
| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权或token过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 3. 接口列表

| 接口名称 | 接口路径 | 请求方法 | 功能描述 |
|----------|----------|----------|----------|
| 获取代金券列表 | `/api/vouchers` | GET | 获取用户代金券列表，支持按状态筛选 |
| 获取代金券详情 | `/api/vouchers/{id}` | GET | 获取单个代金券的详细信息 |

## 4. 详细接口说明

### 4.1 获取代金券列表

#### 4.1.1 接口路径
- `GET /api/vouchers`

#### 4.1.2 功能描述
获取用户已拥有的代金券列表，支持按状态（已领取、已使用、已过期）筛选。

#### 4.1.3 请求参数
| 参数名 | 类型 | 必选 | 位置 | 说明 |
|--------|------|------|------|------|
| status | string | 否 | query | 代金券状态：`received`（已领取）、`used`（已使用）、`expired`（已过期） |
| page | int | 否 | query | 页码，默认1 |
| pageSize | int | 否 | query | 每页数量，默认10 |

#### 4.1.4 响应参数
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "id": "voucher_001",
        "name": "9.9元代金券",
        "faceValue": 9.9,
        "minSpend": 9.91,
        "status": "received",
        "startTime": "2025-12-02",
        "endTime": "2026-11-30 23:59:59",
        "issuer": "外卖券助手",
        "description": "满9.91元可用",
        "createdAt": "2025-12-01 10:00:00"
      }
    ],
    "total": 1,
    "page": 1,
    "pageSize": 10
  }
}
```

#### 4.1.5 字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | string | 代金券ID |
| name | string | 代金券名称 |
| faceValue | number | 面额，单位：元 |
| minSpend | number | 使用门槛，单位：元 |
| status | string | 状态：`received`（已领取）、`used`（已使用）、`expired`（已过期） |
| startTime | string | 有效开始时间，格式：`YYYY-MM-DD` |
| endTime | string | 有效结束时间，格式：`YYYY-MM-DD HH:mm:ss` |
| issuer | string | 发行方 |
| description | string | 简要描述 |
| createdAt | string | 领取时间，格式：`YYYY-MM-DD HH:mm:ss` |

### 4.2 获取代金券详情

#### 4.2.1 接口路径
- `GET /api/vouchers/{id}`

#### 4.2.2 功能描述
获取单个代金券的详细信息，包括使用须知等。

#### 4.2.3 请求参数
| 参数名 | 类型 | 必选 | 位置 | 说明 |
|--------|------|------|------|------|
| id | string | 是 | path | 代金券ID |

#### 4.2.4 响应参数
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "voucher_001",
    "name": "9.9元代金券",
    "type": "cash",
    "faceValue": 9.9,
    "minSpend": 9.91,
    "status": "received",
    "startTime": "2025-12-02",
    "endTime": "2026-11-30 23:59:59",
    "issuer": "外卖券助手",
    "description": "满9.91元可用",
    "useInstructions": [
      "每人限用1张",
      "具体使用规则以券面说明为准"
    ],
    "createdAt": "2025-12-01 10:00:00",
    "usedAt": null
  }
}
```

#### 4.2.5 字段说明
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | string | 代金券ID |
| name | string | 代金券名称 |
| type | string | 类型：`cash`（现金券） |
| faceValue | number | 面额，单位：元 |
| minSpend | number | 使用门槛，单位：元 |
| status | string | 状态：`received`（已领取）、`used`（已使用）、`expired`（已过期） |
| startTime | string | 有效开始时间，格式：`YYYY-MM-DD` |
| endTime | string | 有效结束时间，格式：`YYYY-MM-DD HH:mm:ss` |
| issuer | string | 发行方 |
| description | string | 简要描述 |
| useInstructions | array | 使用须知列表 |
| createdAt | string | 领取时间，格式：`YYYY-MM-DD HH:mm:ss` |
| usedAt | string | 使用时间，格式：`YYYY-MM-DD HH:mm:ss`，未使用则为null |
