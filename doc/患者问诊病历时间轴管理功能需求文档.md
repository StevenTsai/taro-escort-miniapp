# 患者问诊病历时间轴管理功能需求文档

## 1. 文档概述

### 1.1 文档目的
本文档描述了医疗陪护小程序患者问诊病历时间轴管理功能的需求，旨在帮助医生和患者更好地管理和查看患者的问诊历史、治疗记录和检查结果，形成完整的患者健康时间轴。

### 1.2 适用范围
本文档适用于医疗陪护小程序的开发团队，包括产品经理、后端开发、前端开发和测试人员。

### 1.3 术语定义
- **问诊病历**：患者在医院或诊所就诊时产生的医疗记录
- **时间轴**：按照时间顺序展示的患者健康事件序列
- **治疗小结**：医生对患者治疗过程和结果的总结
- **基础病史**：患者的基本健康状况和过往病史
- **影像数据**：包括CT、MRI等医学影像检查结果
- **关键节点**：对患者治疗过程有重要影响的事件

## 2. 功能需求

### 2.1 患者信息管理

#### 2.1.1 基础信息维护
- 支持录入和编辑患者基本信息（姓名、性别、年龄、身高、体重等）
- 支持查看患者基本信息概览

#### 2.1.2 基础病史管理
- 支持录入和编辑患者基础病史
- 包括既往病史、过敏史、家族病史等
- 支持按时间顺序展示病史记录

### 2.2 问诊病历数据输入

#### 2.2.1 治疗小结录入
- 支持医生录入患者每次治疗的小结
- 包括就诊日期、就诊医院、医生姓名、诊断结果、治疗方案等
- 支持富文本编辑，可插入表格、图片等

#### 2.2.2 用药情况管理
- 支持录入患者的用药记录
- 包括药物名称、剂量、用法、用药时间等
- 支持批量导入和导出用药记录

#### 2.2.3 影像数据上传
- 支持上传医学影像数据（CT、MRI、X光片等）
- 支持图片格式（JPG、PNG等）和DICOM格式
- 支持图片预览和下载
- 支持为影像数据添加描述和标签

#### 2.2.4 检查报告管理
- 支持上传和管理各种检查报告（血常规、尿常规、生化检查等）
- 支持PDF、图片等格式
- 支持报告内容预览

### 2.3 时间轴展示

#### 2.3.1 时间轴概览
- 按照时间顺序展示患者的所有健康事件
- 支持按时间范围筛选（如最近3个月、最近1年等）
- 支持按事件类型筛选（如检查、治疗、用药等）

#### 2.3.2 事件详情展示
- 点击时间轴上的事件可查看详细信息
- 包括事件时间、事件类型、详细内容、相关附件等
- 支持查看事件之间的关联关系

#### 2.3.3 关键节点标记
- 支持将重要事件标记为关键节点
- 关键节点在时间轴上突出显示
- 支持快速跳转到关键节点

### 2.4 数据查询与统计

#### 2.4.1 时间轴搜索
- 支持按关键词搜索时间轴上的事件
- 支持按日期范围搜索
- 支持按事件类型搜索

#### 2.4.2 统计分析
- 支持生成患者健康统计报告
- 包括就诊次数统计、用药统计、检查项目统计等
- 支持图表展示统计结果

### 2.5 权限管理

#### 2.5.1 患者权限
- 患者可查看自己的完整时间轴
- 患者可编辑自己的基本信息和基础病史

#### 2.5.2 医生权限
- 医生可查看和编辑其管理的患者的时间轴
- 医生可录入治疗小结、用药记录等专业信息

#### 2.5.3 管理员权限
- 管理员可管理所有患者的时间轴
- 管理员可配置系统参数和用户权限

## 3. 数据模型设计

### 3.1 患者信息表（patient_info）

| 字段名 | 数据类型 | 描述 |
|-------|---------|------|
| id | VARCHAR(36) | 患者ID |
| name | VARCHAR(50) | 患者姓名 |
| gender | INT | 性别（1:男 2:女） |
| age | INT | 年龄 |
| height | DECIMAL(5,2) | 身高（cm） |
| weight | DECIMAL(5,2) | 体重（kg） |
| phone | VARCHAR(20) | 联系电话 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.2 基础病史表（patient_history）

| 字段名 | 数据类型 | 描述 |
|-------|---------|------|
| id | VARCHAR(36) | 病史ID |
| patient_id | VARCHAR(36) | 患者ID |
| type | INT | 病史类型（1:既往病史 2:过敏史 3:家族病史） |
| content | TEXT | 病史内容 |
| record_date | DATE | 记录日期 |
| created_at | DATETIME | 创建时间 |

### 3.3 时间轴事件表（timeline_event）

| 字段名 | 数据类型 | 描述 |
|-------|---------|------|
| id | VARCHAR(36) | 事件ID |
| patient_id | VARCHAR(36) | 患者ID |
| event_date | DATE | 事件日期 |
| event_type | INT | 事件类型（1:就诊 2:检查 3:用药 4:治疗） |
| title | VARCHAR(100) | 事件标题 |
| content | TEXT | 事件内容 |
| is_key_node | INT | 是否关键节点（0:否 1:是） |
| created_by | VARCHAR(36) | 创建人ID |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.4 附件表（attachment）

| 字段名 | 数据类型 | 描述 |
|-------|---------|------|
| id | VARCHAR(36) | 附件ID |
| event_id | VARCHAR(36) | 事件ID |
| file_name | VARCHAR(200) | 文件名 |
| file_path | VARCHAR(500) | 文件路径 |
| file_type | VARCHAR(50) | 文件类型（如image/jpeg、application/pdf等） |
| file_size | BIGINT | 文件大小（字节） |
| upload_by | VARCHAR(36) | 上传人ID |
| upload_time | DATETIME | 上传时间 |

### 3.5 用药记录表（medication_record）

| 字段名 | 数据类型 | 描述 |
|-------|---------|------|
| id | VARCHAR(36) | 用药记录ID |
| event_id | VARCHAR(36) | 事件ID |
| drug_name | VARCHAR(100) | 药物名称 |
| dosage | VARCHAR(50) | 剂量 |
| usage | VARCHAR(100) | 用法 |
| start_date | DATE | 开始用药日期 |
| end_date | DATE | 结束用药日期 |
| created_at | DATETIME | 创建时间 |

## 4. 界面设计

### 4.1 患者信息概览页面

#### 4.1.1 布局
- 顶部显示患者基本信息（姓名、性别、年龄、身高、体重）
- 中部显示关键节点概览
- 底部显示最近的健康事件预览

#### 4.1.2 交互
- 点击基本信息可进入编辑页面
- 点击关键节点可跳转到时间轴对应位置
- 点击查看全部可进入完整时间轴页面

### 4.2 时间轴页面

#### 4.2.1 布局
- 顶部显示筛选条件（时间范围、事件类型）
- 中部显示时间轴主体，按时间倒序排列
- 每个事件条目显示事件日期、类型、标题和简要内容
- 关键节点用特殊样式标记

#### 4.2.2 交互
- 点击事件条目可展开查看详细信息
- 支持滑动加载更多历史事件
- 支持点击关键节点快速定位

### 4.3 事件详情页面

#### 4.3.1 布局
- 顶部显示事件基本信息（日期、类型、标题）
- 中部显示事件详细内容
- 底部显示相关附件列表

#### 4.3.2 交互
- 支持编辑事件内容
- 支持上传和下载附件
- 支持标记/取消标记关键节点

### 4.4 数据录入页面

#### 4.4.1 治疗小结录入
- 表单式布局，包括就诊日期、医院、医生、诊断结果、治疗方案等字段
- 支持富文本编辑治疗详细内容
- 支持上传相关检查报告和影像数据

#### 4.4.2 用药记录录入
- 支持录入药物名称、剂量、用法等信息
- 支持批量录入多条用药记录
- 支持从常用药物库选择药物

## 5. 技术实现要点

### 5.1 前端实现

#### 5.1.1 时间轴组件
- 使用Taro框架的列表组件实现时间轴布局
- 支持事件的展开/折叠
- 实现关键节点的特殊样式
- 支持滑动加载更多数据

#### 5.1.2 富文本编辑
- 集成富文本编辑器，支持文本格式化、表格、图片插入等功能
- 确保富文本内容在不同设备上的正确显示

#### 5.1.3 文件上传与预览
- 支持图片和文件的上传
- 实现图片预览功能
- 支持PDF等文件的在线预览

### 5.2 后端实现

#### 5.2.1 数据存储
- 使用关系型数据库存储结构化数据
- 使用对象存储服务（如OSS）存储影像数据和文件
- 建立合理的索引，提高查询效率

#### 5.2.2 API设计
- 采用RESTful风格设计API接口
- 实现数据的增删改查功能
- 支持分页查询和筛选

#### 5.2.3 权限控制
- 基于角色的权限控制机制
- 确保患者只能查看自己的信息
- 医生只能查看和编辑其管理的患者信息

### 5.3 数据安全

#### 5.3.1 数据加密
- 患者敏感信息加密存储
- 传输过程使用HTTPS加密

#### 5.3.2 访问控制
- 严格的身份验证和授权机制
- 记录所有数据操作日志
- 定期备份数据

## 6. 非功能需求

### 6.1 性能需求
- 时间轴页面加载时间不超过2秒
- 文件上传速度不低于1MB/s
- 支持同时在线用户数不少于1000人

### 6.2 可用性需求
- 系统可用性不低于99.5%
- 支持7×24小时运行
- 出现故障时恢复时间不超过1小时

### 6.3 可扩展性需求
- 支持后续增加新的事件类型
- 支持与医院HIS系统对接
- 支持大数据量的存储和查询

### 6.4 用户体验需求
- 界面简洁直观，操作便捷
- 支持移动端和桌面端访问
- 提供清晰的操作指引和帮助文档
