# 患者问诊病历时间轴管理功能-前端页面设计文档

## 1. 文档概述

### 1.1 文档目的
本文档是患者问诊病历时间轴管理功能的前端页面设计文档，基于需求文档进行细化，提供UI设计、交互设计、前端技术实现等具体实现细节，为前端开发团队提供直接的技术指导。

### 1.2 设计范围
本文档覆盖患者问诊病历时间轴管理功能的前端模块，包括：
- 患者信息管理模块（创建患者页面、患者基础信息更新页面）
- 问诊病历数据输入模块（治疗小结录入页面、用药事件录入页面、检查事件录入页面）
- 时间轴展示模块（患者信息概览页面、时间轴页面、事件详情页面）

- 《患者问诊病历时间轴管理功能需求文档》
- Taro框架文档
- 微信小程序开发文档

├─────────────────────────────────────────────────┤
│ 姓名：[输入框]                                   │
│ 性别：[单选框] 男 □ 女 □                          │
│ 年龄：[数字输入框]                                │
│ 身高：[数字输入框] cm                              │
│ 体重：[数字输入框] kg                              │
│ 联系电话：[输入框]                                │
├─────────────────────────────────────────────────┤
│ 基础病史：                                       │
│ 既往病史：[富文本编辑器]                          │
│ 过敏史：[富文本编辑器]                            │
│ 家族病史：[富文本编辑器]                          │
├─────────────────────────────────────────────────┤

#### 2.1.2 组件规格
- 头部区域：使用导航栏组件，背景色#1890ff，文字颜色白色，字体大小18px，高度44px
- 表单区域：使用表单组件，每个字段包含标签和输入控件，标签右对齐，文字颜色#333333，字体大小15px
- 姓名输入框：单行文本输入，最大长度50字符，必填项，占位符"请输入患者姓名"
- 性别选择：使用单选框组件，默认选中"男"，选项间距16px，选中状态使用蓝色#1890ff
- 年龄输入框：数字输入，范围1-120，必填项，占位符"请输入年龄"
- 身高输入框：数字输入，支持两位小数，范围50-250，占位符"请输入身高"
- 体重输入框：数字输入，支持一位小数，范围10-200，占位符"请输入体重"
- 联系电话：电话格式输入，支持11位手机号码，必填项，占位符"请输入联系电话"
- 基础病史：使用富文本编辑器，支持文本格式化（加粗、斜体、下划线）、换行、段落和图片插入
- 保存按钮：使用主按钮样式，蓝色#1890ff，圆角4px，字体大小16px，文字颜色白色，高度44px，宽度100%

#### 2.1.3 视觉设计细节
- 表单字段间距：上下间距16px，左右内边距16px
- 输入框样式：边框1px solid #e8e8e8，圆角4px，高度40px，内边距8px，字体大小14px
- 输入框聚焦样式：边框颜色#1890ff，阴影0 0 0 2px rgba(24, 144, 255, 0.2)
- 错误提示样式：文字颜色#ff4d4f，字体大小12px，距离输入框底部4px，左侧与输入框对齐
- 富文本编辑器：
  - 高度：120px，带垂直滚动条
  - 工具栏：包含加粗、斜体、下划线、换行、图片插入等按钮
  - 背景色：#ffffff，边框1px solid #e8e8e8，圆角4px
- 页面背景色：#ffffff
- 响应式设计：
  - 大屏幕设备：表单字段左右内边距32px，输入框宽度自适应
  - 小屏幕设备：表单字段左右内边距16px，输入框宽度100%

#### 2.1.4 表单验证设计
- **实时验证**：用户输入时立即验证格式，显示即时反馈
- **提交验证**：点击保存按钮时进行完整验证
- **验证规则**：
  - 姓名：必填，长度1-50字符
  - 性别：必填，只能选择男或女
  - 年龄：必填，1-120之间的整数
  - 身高：可选，50-250之间的数字，最多两位小数
  - 体重：可选，10-200之间的数字，最多一位小数
  - 联系电话：必填，11位手机号码格式
  - 病史内容：可选，但输入时要求有意义的文本内容

#### 2.1.5 富文本编辑器功能设计
- **基本编辑功能**：
  - 文本格式化：加粗、斜体、下划线
  - 段落控制：换行、段落间距调整
  - 图片处理：支持图片插入、预览、删除
- **图片上传功能**：
  - 支持JPG、PNG、GIF格式，单张图片最大5MB
  - 上传进度显示：百分比进度条
  - 图片预览：点击图片可放大查看
- **内容限制**：
  - 单个病史内容最大5000字符
  - 支持HTML格式存储和渲染
  - 自动过滤恶意脚本和标签

##### 设计原则
- **复用性**：完全复用治疗小结、检查事件、用药事件的录入页面布局和组件规格
- **一致性**：保持与新增页面相同的交互模式和视觉风格
- **效率**：减少用户学习成本，提高操作效率

##### 具体实现
- **页面标题**：将"新增"改为"编辑"（如："编辑检查事件"）
- **数据加载**：进入页面时自动加载当前事件的现有数据并填充到表单中
- **操作逻辑**：保存后更新现有事件数据并返回时间轴页面
- **附件管理**：支持在原有附件基础上添加或删除附件

#### 2.5.2 与新增页面的关系
- **布局复用**：完全复用对应事件类型的新增页面布局
- **组件复用**：复用所有表单字段、验证规则和交互组件
- **数据差异**：编辑页面加载现有数据，新增页面为空表单
- **保存逻辑**：编辑页面调用更新API，新增页面调用创建API

## 3. 交互设计

### 3.1 创建患者页面交互

#### 3.1.1 表单输入交互
- **触发条件**：用户在输入框中输入内容或选择下拉选项

  1. 用户输入内容或选择选项时，实时验证格式

  3. 输入格式正确时，隐藏错误提示

function MedicationEventForm() {
  const [formData, setFormData] = useState({

        drug_name: '',
        dosage: '',
        usage: '',
        start_date: '',
        end_date: ''
      }]
    });

  // 删除用药记录
  const removeMedication = (index) => {
    Taro.showModal({
      title: '确认删除',
      content: '确定要删除这条用药记录吗？',
      success: (res) => {
        if (res.confirm) {
          const newMedications = [...formData.medications];

          setFormData({...formData, medications: newMedications});
        }

    newMedications[index][field] = value;

  };

  const validateForm = () => {
    const newErrors = {};
    
    // 验证用药日期
    if (!formData.event_date) {
      newErrors.event_date = '用药日期不能为空';
    }
    
    // 验证用药记录
    const medicationErrors = [];
    formData.medications.forEach((med, index) => {
      const medError = {};
      if (!med.drug_name) medError.drug_name = '药物名称不能为空';
      if (!med.dosage) medError.dosage = '剂量不能为空';
      if (!med.usage) medError.usage = '用法不能为空';
      if (!med.start_date) medError.start_date = '开始日期不能为空';
      if (med.end_date && med.end_date < med.start_date) {
        medError.end_date = '结束日期不能早于开始日期';
      }
      if (Object.keys(medError).length > 0) {
        medicationErrors[index] = medError;
      }
    });
    
    if (medicationErrors.length > 0) {
      newErrors.medications = medicationErrors;
    }
    
    setErrors(newErrors);
    return Object.keys(newErrors).length === 0;
  };

## 6. UI/UX优化建议（专业UI设计师审查）

作为专业UI设计师，我对当前的时间轴前端设计文档进行了全面审查，并提出以下优化建议：

### 6.1 视觉设计优化

#### 6.1.1 色彩系统优化
- **事件类型颜色增强**：当前事件类型使用了不同颜色标记（治疗小结#1890ff、检查事件#52c41a、用药事件#faad14），建议增加颜色的饱和度和区分度，同时为每种事件类型添加对应的图标，提高视觉识别效率
  - 治疗小结：保留蓝色#1890ff，添加📋图标
  - 检查事件：调整为更深的绿色#389e0d，添加🔬图标
  - 用药事件：调整为更深的橙色#d48806，添加💊图标

- **病史类型颜色优化**：当前病史类型颜色（既往病史#8c8c8c、过敏史#ff4d4f、家族病史#1890ff）中，既往病史的灰色对比度较低，建议调整为#595959，提高可读性

- **背景色分层**：为时间轴页面添加轻微的背景色分层，主内容区使用白色#ffffff，筛选区域使用浅灰色#f5f5f5，创建视觉层次感

#### 6.1.2 排版系统优化
- **字体层级明确化**：建立清晰的字体层级结构
  - 页面标题：18px，加粗，#333333
  - 区域标题：16px，加粗，#333333
  - 事件标题：15px，#333333
  - 事件内容：14px，#666666
  - 日期时间：12px，#999999

- **行高优化**：增加事件内容的行高至1.6，提高长文本的可读性

- **文本截断策略**：明确所有文本内容的截断规则，超过3行的事件内容显示"展开"按钮，避免信息丢失

#### 6.1.3 间距系统优化
- **事件条目间距**：增加事件条目之间的间距至16px，提高视觉呼吸感
- **内边距统一**：统一所有卡片组件的内边距为16px，创建一致的视觉体验
- **日期线与内容间距**：增加日期线与事件内容之间的水平间距至12px，提高布局清晰度

### 6.2 交互体验优化

#### 6.2.1 筛选功能优化
- **筛选条件布局**：当前水平布局在小屏幕上可能拥挤，建议采用弹性布局，在屏幕宽度不足时自动调整为垂直堆叠
- **实时筛选反馈**：添加筛选条件应用动画（如平滑过渡效果），让用户感知筛选操作的执行
- **筛选结果统计**：在筛选区域下方显示"共找到X条记录"的统计信息，提高用户对数据量的认知

#### 6.2.2 事件条目交互
- **悬停效果增强**：为事件条目添加悬停效果（阴影加深、轻微上浮），提高可点击性的视觉暗示
- **点击反馈优化**：将点击时的背景色从#f5f5f5调整为更明显的#e6f7ff（蓝色系），与主题色保持一致
- **快速操作菜单**：为每条事件添加右键菜单或滑动操作，支持快速编辑/删除功能

#### 6.2.3 加载状态优化
- **骨架屏设计**：在数据加载时使用骨架屏替代简单的"加载中"文字，提供更好的视觉反馈
- **无限滚动优化**：在滚动加载更多时，添加加载指示器（如进度条或旋转动画），让用户了解加载进度
- **空状态设计**：优化无数据时的显示界面，提供更友好的提示信息和引导操作（如"暂无数据，点击添加第一条记录"）

### 6.3 响应式设计优化

#### 6.3.1 移动设备适配
- **筛选区域折叠**：在移动设备上，将筛选条件默认折叠，点击"筛选"按钮展开，节省屏幕空间
- **单列布局优化**：确保在小屏幕设备上，事件条目完全以单列显示，避免内容拥挤
- **触摸目标优化**：增大可点击元素的尺寸（至少44x44px），提高移动端触摸操作的准确性

#### 6.3.2 大屏幕适配
- **多列布局**：在大屏幕设备上（宽度≥1200px），考虑采用双列时间轴布局，提高信息密度
- **侧边栏扩展**：在超大屏幕设备上，可以将筛选条件固定在左侧侧边栏，主内容区显示时间轴

### 6.4 可访问性优化

#### 6.4.1 颜色对比度
- **增强文本对比度**：确保所有文本与背景色的对比度符合WCAG AA标准（至少4.5:1）
- **色弱友好设计**：为颜色标记添加纹理或图案辅助，确保色弱用户也能区分不同事件类型

#### 6.4.2 键盘导航
- **支持完全键盘操作**：确保所有功能都可以通过键盘访问（Tab键导航、Enter键确认等）
- **焦点状态明确**：为所有可交互元素添加清晰的焦点状态样式（如蓝色边框或阴影）

#### 6.4.3 屏幕阅读器支持
- **语义化HTML**：使用正确的HTML语义标签（如`<header>`, `<main>`, `<section>`）
- **ARIA属性**：为复杂组件添加适当的ARIA属性，提高屏幕阅读器的可访问性

### 6.5 代码实现优化建议

#### 6.5.1 组件化设计
- **时间轴组件抽象**：将时间轴功能抽象为可复用组件，支持自定义事件类型、颜色和渲染方式
- **筛选组件封装**：封装通用的筛选组件，支持多种筛选条件类型（下拉选择、日期范围、搜索框等）

#### 6.5.2 性能优化
- **虚拟列表实现**：当时间轴事件数量较多时，采用虚拟列表技术，只渲染可见区域的事件，提高页面性能
- **图片懒加载**：对事件中包含的图片资源实现懒加载，减少初始加载时间

#### 6.5.3 状态管理
- **统一状态管理**：使用Redux或Context API统一管理时间轴的筛选条件、当前页码等状态，提高代码可维护性
- **本地存储优化**：将用户的筛选偏好存储在localStorage中，下次访问时自动恢复
