- 富文本编辑器：
  - 高度：120px，带垂直滚动条
  - 工具栏：包含加粗、斜体、下划线、换行、图片插入等按钮

  - 大屏幕设备：表单字段左右内边距32px，输入框宽度自适应
  - 小屏幕设备：表单字段左右内边距16px，输入框宽度100%

#### 2.1.4 表单验证设计
- **实时验证**：用户输入时立即验证格式，显示即时反馈
- **提交验证**：点击保存按钮时进行完整验证
- **验证规则**：
  - 姓名：必填，长度1-50字符
  - 性别：必填，只能选择男或女
  - 年龄：必填，1-120之间的整数
  - 身高：可选，50-250之间的数字，最多两位小数
  - 体重：可选，10-200之间的数字，最多一位小数
  - 联系电话：必填，11位手机号码格式
  - 病史内容：可选，但输入时要求有意义的文本内容

#### 2.1.5 富文本编辑器功能设计
- **基本编辑功能**：
  - 文本格式化：加粗、斜体、下划线
  - 段落控制：换行、段落间距调整
  - 图片处理：支持图片插入、预览、删除
- **图片上传功能**：
  - 支持JPG、PNG、GIF格式，单张图片最大5MB
  - 上传进度显示：百分比进度条
  - 图片预览：点击图片可放大查看
- **内容限制**：
  - 单个病史内容最大5000字符
  - 支持HTML格式存储和渲染
  - 自动过滤恶意脚本和标签

## 3. 交互设计

### 3.1 创建患者页面交互

#### 3.1.1 表单输入交互
- **触发条件**：用户在输入框中输入内容或选择下拉选项
- **交互流程**：
  1. 用户输入内容或选择选项时，实时验证格式
  2. 输入格式错误时，显示错误提示信息
  3. 输入格式正确时，隐藏错误提示
- **视觉反馈**：错误输入框/选择框显示红色边框，正确输入框/选择框显示绿色边框

#### 3.1.2 富文本编辑器交互
- **触发条件**：用户点击富文本编辑器
- **交互流程**：
  1. 打开富文本编辑面板
  2. 用户进行文本编辑操作
  3. 编辑完成后，点击保存按钮
  4. 编辑器内容保存到表单中
- **视觉反馈**：编辑面板覆盖部分屏幕，保存按钮高亮显示

#### 3.1.3 保存操作交互
- **触发条件**：用户点击保存按钮
- **交互流程**：
  1. 系统进行表单验证
  2. 验证通过，显示加载动画
  3. 调用API保存患者信息
  4. 保存成功，跳转至患者信息概览页面
  5. 保存失败，显示错误提示
- **视觉反馈**：加载动画覆盖整个页面，成功/失败提示使用Toast组件

### 3.2 时间轴页面交互

#### 3.2.1 事件展开/折叠
- **触发条件**：点击时间轴上的事件条目
- **交互流程**：
  1. 用户点击事件条目
  2. 事件条目展开，显示详细内容和附件列表
  3. 再次点击，事件条目折叠

#### 3.2.2 滑动加载更多
- **触发条件**：用户滑动到时间轴底部
- **交互流程**：
  1. 用户滑动页面到底部

- **触发条件**：点击筛选条件下拉框
- **交互流程**：
  1. 用户点击时间范围或事件类型下拉框
  2. 弹出选择面板

  4. 点击确认按钮，更新时间轴内容
- **视觉反馈**：筛选条件更新后，时间轴内容重新加载

## 4. API接口设计

### 4.1 患者信息管理接口

#### 4.1.1 创建患者
- **接口路径**：`/api/patients`
- **请求方法**：`POST`
- **请求头**：
  - `Content-Type`: `application/json`
  - `Authorization`: `Bearer {token}`
  - `X-Request-ID`: 可选，请求唯一标识符，用于日志追踪
- **权限要求**：医生、护士、管理员角色
- **请求参数**：
```json
{
  "name": "张三",
  "gender": 1,
  "age": 65,
  "height": 175.00,
  "weight": 70.00,
  "phone": "13800138000",
  "histories": [
    {
      "type": 1,
      "content": "高血压病史10年",
      "record_date": "2024-01-01"
    },
    {
      "type": 2,
      "content": "青霉素过敏",
      "record_date": "2024-01-01"
    }
  ]
}
```
- **参数说明**：
  - `name`：患者姓名（必填，字符串，最大长度50）
  - `gender`：性别（必填，数字，1-男，2-女）
  - `age`：年龄（必填，数字，1-120）
  - `height`：身高（可选，数字，单位cm，保留两位小数，范围50-250）
  - `weight`：体重（可选，数字，单位kg，保留一位小数，范围10-200）
  - `phone`：联系电话（必填，字符串，11位手机号码，格式：1[3-9]\d{9}）
  - `histories`：基础病史列表（可选，数组，最多10个病史条目）
    - `type`：病史类型（必填，数字，1-既往病史，2-过敏史，3-家族病史）
    - `content`：病史内容（必填，字符串，最大长度5000，支持HTML格式）
    - `record_date`：记录日期（必填，字符串，格式YYYY-MM-DD，不能超过当前日期）

- **响应示例**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": "pat_123",
    "name": "张三",
    "gender": 1,
    "age": 65,
    "height": 175.00,
    "weight": 70.00,
    "phone": "13800138000",
    "created_at": "2024-05-20 14:30:00",
    "histories_count": 2
  },
  "request_id": "req_1234567890"
}
```

- **错误响应**：
  - **参数错误**：
  ```json
  {
    "code": 400,
    "msg": "参数错误",
    "data": {
      "errors": {
        "name": "患者姓名不能为空",
        "phone": "请输入正确的手机号码",
        "age": "年龄必须在1-120之间",
        "histories.0.record_date": "记录日期不能超过当前日期"
      }
    },
    "request_id": "req_1234567890"
  }
  ```
  
  - **权限不足**：
  ```json
  {
    "code": 403,
    "msg": "权限不足，无法创建患者信息",
    "request_id": "req_1234567890"
  }
  ```
  
  - **服务器错误**：
  ```json
  {
    "code": 500,
    "msg": "服务器内部错误",
    "request_id": "req_1234567890"
  }
  ```

- **设计说明**：
  - 接口支持批量创建患者基础病史，减少后续API调用次数
  - 病史内容支持HTML格式，便于富文本编辑器内容的存储和渲染
  - 所有必填字段都有明确的格式和范围限制
  - 错误响应包含详细的错误信息，便于前端展示和调试
  - 支持请求ID追踪，便于问题排查和日志分析
  - 使用事务确保患者信息和病史数据的一致性

#### 4.1.2 获取患者信息
- **接口路径**：`/api/patients/{id}`
- **请求方法**：`GET`
- **请求头**：
  - `Authorization`: `Bearer {token}`
- **请求参数**：
  - `id`: 患者ID（路径参数，必填）
- **响应示例**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "id": "pat_123",
    "name": "张三",
    "gender": 1,
    "age": 65,
    "height": 175.00,
    "weight": 70.00,
    "phone": "13800138000",
```

```javascript
// 创建患者表单组件
function CreatePatientForm() {
  const [formData, setFormData] = useState({
    name: '',
    gender: 1,
    age: '',
    height: '',
    weight: '',
    phone: '',
    histories: [
      { type: 1, content: '', record_date: '' },

    ]
  });
  const [errors, setErrors] = useState({});

    if (!formData.name) newErrors.name = '患者姓名不能为空';

      newErrors.phone = '联系电话不能为空';

    if (!formData.age) {

    } else if (formData.age < 1 || formData.age > 120) {
      newErrors.age = '年龄必须在1-120之间';
    }
    setErrors(newErrors);
    return Object.keys(newErrors).length === 0;
  };

  const handleSubmit = async () => {
    if (!validateForm()) return;
    
    setLoading(true);
    try {
      const response = await post('/api/patients', formData);
      if (response.code === 200) {
        Taro.showToast({ title: '创建成功', icon: 'success' });
        setTimeout(() => {
          Taro.navigateTo({ url: `/pages/patientDetail/patientDetail?id=${response.data.id}` });

      }
    } catch (error) {
      Taro.showToast({ title: '创建失败', icon: 'none' });
    } finally {
      setLoading(false);
    }
  };

  return (
    <View className="create-patient-form">
      {/* 表单字段 */}
      <FormItem 
        label="姓名" 
        error={errors.name}
        onChange={(value) => setFormData({...formData, name: value})}

      {/* 富文本编辑器 */}

            setFormData({...formData, histories: newHistories});
          }}
        />
      </View>

      <Button 
        className="submit-button"

  );
}
```

#### 6.2.1 创建患者接口实现
```javascript
// Node.js Express示例
app.post('/api/patients', async (req, res) => {
  try {
    // 参数验证
    const { error } = validatePatient(req.body);
    if (error) {
      return res.status(400).json({
        code: 400,
        msg: '参数错误',
        data: { errors: error.details.reduce((acc, curr) => {
          acc[curr.context.key] = curr.message;
          return acc;
        }, {})}
      });
    }

    // 开始事务
    const connection = await pool.getConnection();
    await connection.beginTransaction();

    try {
      // 创建患者信息
      const patientId = uuidv4();
      await connection.query(
        'INSERT INTO patient_info (id, name, gender, age, height, weight, phone) VALUES (?, ?, ?, ?, ?, ?, ?)',
        [patientId, req.body.name, req.body.gender, req.body.age, req.body.height, req.body.weight, req.body.phone]
      );

      // 创建基础病史
      if (req.body.histories && req.body.histories.length > 0) {
        for (const history of req.body.histories) {
          if (history.content) {
            await connection.query(
              'INSERT INTO patient_history (id, patient_id, type, content, record_date) VALUES (?, ?, ?, ?, ?)',
              [uuidv4(), patientId, history.type, history.content, history.record_date]
            );
          }
        }
      }

      // 提交事务
      await connection.commit();
      connection.release();

      // 返回结果
      res.json({
        code: 200,
        msg: 'success',
        data: { id: patientId, ...req.body }
      });
    } catch (err) {
      await connection.rollback();
      connection.release();
      throw err;
    }
  } catch (err) {
    res.status(500).json({
      code: 500,
      msg: '服务器错误',
      data: { error: err.message }
    });
  }
});
```
