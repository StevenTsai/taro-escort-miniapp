# 医疗陪护小程序 - 视频录播功能设计文档

## 1. 需求分析

### 1.1 功能需求分析

#### 1.1.1 课程列表功能
- **功能描述**：展示医疗相关课程列表，支持分类筛选和关键词搜索
- **用户场景**：用户进入课程列表页，浏览所有课程，或通过分类和搜索查找特定课程
- **功能点**：
  - 课程列表展示（包含课程封面、标题、讲师、价格、时长、观看次数等信息）
  - 课程分类筛选（支持切换不同分类）
  - 关键词搜索（支持实时搜索）
  - 下拉刷新和上拉加载更多

#### 1.1.2 课程详情功能
- **功能描述**：展示课程的详细信息，包括课程介绍、课程大纲、讲师信息、用户评价等
- **用户场景**：用户点击课程列表中的某个课程，进入课程详情页查看详细信息
- **功能点**：
  - 课程基本信息展示（封面、标题、价格、时长、观看次数、评分等）
  - 课程介绍（图文混排）
  - 课程大纲（展示课程章节和视频列表）
  - 讲师信息（讲师头像、姓名、头衔、简介等）
  - 用户评价（展示用户评价列表，支持评分）
  - 收藏/取消收藏课程
  - 购买课程（如果是付费课程）

#### 1.1.3 视频播放功能
- **功能描述**：播放课程视频，支持播放控制、进度记录等
- **用户场景**：用户在课程详情页点击某个视频，进入视频播放页观看视频
- **功能点**：
  - 视频播放/暂停
  - 进度条控制（支持拖动定位）
  - 音量控制
  - 全屏切换
  - 播放速度调节
  - 自动记录观看进度
  - 视频封面展示
  - 视频标题和简介展示
  - 相关推荐（展示相关课程或视频）

#### 1.1.4 我的课程功能
- **功能描述**：展示用户已购买或已收藏的课程
- **用户场景**：用户进入我的课程页，查看自己购买或收藏的课程
- **功能点**：
  - 我的课程列表（已购买的课程）
  - 收藏课程列表
  - 课程分类筛选
  - 课程搜索

#### 1.1.5 观看历史功能
- **功能描述**：展示用户的观看历史记录
- **用户场景**：用户进入观看历史页，查看自己之前观看过的视频
- **功能点**：
  - 观看历史列表（包含视频封面、标题、观看进度、观看时间等）
  - 清空观看历史
  - 继续观看（从上次观看进度继续播放）

### 1.2 非功能需求分析

#### 1.2.1 性能需求
- 视频播放流畅度：支持720p高清视频流畅播放
- 页面加载速度：首屏加载时间不超过2秒
- 响应时间：交互操作响应时间不超过500ms

#### 1.2.2 兼容性需求
- 支持微信小程序基础库版本2.10.0及以上
- 支持主流手机设备（iOS 12.0+，Android 6.0+）

#### 1.2.3 安全性需求
- 视频播放地址加密，防止盗链
- 权限控制严格，防止未授权访问
- 用户数据安全，保护用户隐私

#### 1.2.4 可用性需求
- 界面简洁明了，操作流程顺畅
- 提供清晰的错误提示
- 支持离线缓存（可选）

## 2. 技术选型

### 2.1 视频播放器选型
- **方案**：使用微信小程序原生 `video` 组件
- **理由**：
  - 微信小程序原生支持，兼容性好
  - 功能完善，支持播放控制、进度条、音量控制、全屏等
  - 性能优化好，播放流畅
  - 支持视频加密播放

### 2.2 视频存储与分发
- **方案**：使用云存储（如腾讯云COS）+ CDN加速
- **理由**：
  - 云存储容量大，支持高并发访问
  - CDN加速可以提高视频加载速度，减少延迟
  - 支持视频加密和防盗链
  - 提供完善的API接口，方便集成

### 2.3 视频加密方案
- **方案**：使用HLS加密（m3u8格式）
- **理由**：
  - 支持视频分段加密，安全性高
  - 微信小程序原生支持m3u8格式
  - 支持自适应码率，根据网络情况自动调整视频质量

### 2.4 状态管理方案
- **方案**：使用Zustand（与现有项目保持一致）
- **理由**：
  - 轻量级，性能好
  - API简单易用
  - 支持异步操作
  - 与React和Taro兼容良好

### 2.5 UI组件选型
- **方案**：使用NutUI-React-Taro（与现有项目保持一致）
- **理由**：
  - 与Taro框架兼容良好
  - 组件丰富，样式美观
  - 支持主题定制
  - 文档完善，社区活跃

## 3. 页面设计

### 3.1 课程列表页设计

#### 3.1.1 UI设计
- **整体布局**：
  - 顶部：搜索栏 + 分类标签
  - 中部：课程列表（网格布局，2列）
  - 底部：无（使用小程序原生底部导航）
- **课程卡片设计**：
  - 顶部：课程封面图（比例16:9）
  - 中部：课程标题（2行，超出省略）
  - 底部：讲师名称 + 价格 + 时长 + 观看次数
- **颜色方案**：
  - 主色调：#1989fa（与现有项目保持一致）
  - 辅助色：#f0f0f0（背景色）
  - 文字颜色：#333333（主文字），#666666（辅助文字），#ff4444（价格）

#### 3.1.2 交互设计
- **分类切换**：点击分类标签，切换显示对应分类的课程
- **搜索功能**：在搜索栏输入关键词，实时显示搜索结果
- **下拉刷新**：下拉页面，刷新课程列表
- **上拉加载更多**：上拉页面，加载更多课程
- **课程点击**：点击课程卡片，进入课程详情页

### 3.2 课程详情页设计

#### 3.2.1 UI设计
- **整体布局**：
  - 顶部：课程基本信息（封面、标题、价格、时长、观看次数、评分等）
  - 中部：
    - 课程介绍（图文混排）
    - 课程大纲（可折叠展开）
    - 讲师信息
    - 用户评价
  - 底部：操作按钮（收藏/取消收藏、购买/立即观看）
- **课程大纲设计**：
  - 章节标题（可折叠展开）
  - 视频列表（包含视频标题、时长、是否免费标识）
- **讲师信息设计**：
  - 讲师头像（圆形）
  - 讲师姓名和头衔
  - 讲师简介

#### 3.2.2 交互设计
- **收藏/取消收藏**：点击收藏按钮，切换收藏状态
- **购买/立即观看**：
  - 如果是免费课程：点击"立即观看"，进入视频播放页
  - 如果是付费课程：
    - 未购买：点击"购买"，进入支付流程
    - 已购买：点击"立即观看"，进入视频播放页
  - 如果是会员课程：
    - 非会员：点击"立即观看"，提示开通会员
    - 会员：点击"立即观看"，进入视频播放页
- **课程大纲展开/折叠**：点击章节标题，展开/折叠该章节下的视频列表
- **视频点击**：点击视频列表中的某个视频，进入视频播放页

### 3.3 视频播放页设计

#### 3.3.1 UI设计
- **整体布局**：
  - 顶部：视频播放器（占满屏幕宽度，高度自适应）
  - 中部：视频标题和简介
  - 底部：相关推荐（课程列表）
- **视频播放器控制栏设计**：
  - 播放/暂停按钮
  - 进度条（显示当前播放时间和总时长）
  - 音量控制按钮
  - 全屏切换按钮
  - 播放速度调节按钮

#### 3.3.2 交互设计
- **播放/暂停**：点击播放/暂停按钮，切换视频播放状态
- **进度控制**：拖动进度条，跳转到指定时间点
- **音量控制**：点击音量按钮，切换静音状态；拖动音量滑块，调节音量大小
- **全屏切换**：点击全屏按钮，切换全屏/非全屏状态
- **播放速度调节**：点击播放速度按钮，选择播放速度（0.5x, 1.0x, 1.5x, 2.0x）
- **返回**：点击左上角返回按钮，返回上一页

### 3.4 我的课程页设计

#### 3.4.1 UI设计
- **整体布局**：
  - 顶部：搜索栏 + 标签页（我的课程/收藏课程）
  - 中部：课程列表（网格布局，2列）
  - 底部：无（使用小程序原生底部导航）

#### 3.4.2 交互设计
- **标签页切换**：点击标签页，切换显示"我的课程"或"收藏课程"
- **搜索功能**：在搜索栏输入关键词，实时显示搜索结果
- **下拉刷新**：下拉页面，刷新课程列表
- **上拉加载更多**：上拉页面，加载更多课程
- **课程点击**：点击课程卡片，进入课程详情页

### 3.5 观看历史页设计

#### 3.5.1 UI设计
- **整体布局**：
  - 顶部：标题 + 清空历史按钮
  - 中部：观看历史列表（列表布局）
  - 底部：无（使用小程序原生底部导航）
- **历史记录项设计**：
  - 左侧：视频封面图（比例16:9）
  - 右侧：

    - 观看进度条 + 进度百分比

#### 3.5.2 交互设计
- **清空历史**：点击右上角"清空历史"按钮，清空所有观看历史
- **继续观看**：点击历史记录项，进入视频播放页，从上次观看进度继续播放
- **下拉刷新**：下拉页面，刷新观看历史列表

## 4. 接口设计

### 4.1 课程相关接口

#### 4.1.1 获取课程列表
- **接口地址**：`/api/courses`
- **请求方法**：GET
- **请求参数**：
  | 参数名 | 类型 | 必填 | 描述 |
  |-------|------|------|------|
  | page | number | 否 | 页码，默认1 |
  | pageSize | number | 否 | 每页数量，默认10 |
  | categoryId | string | 否 | 分类ID，默认所有分类 |
  | keyword | string | 否 | 搜索关键词 |
- **响应格式**：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {
      "list": [
        {
          "id": "course_123",
          "title": "医疗陪护基础知识",
          "coverImage": "https://example.com/cover.jpg",
          "teacherName": "张医生",
          "teacherAvatar": "https://example.com/teacher.jpg",
          "type": "free",
          "price": 0,
          "originalPrice": 0,
          "duration": 3600,
          "viewCount": 1000,
          "rating": 4.8
        }
      ],
      "total": 100,
      "page": 1,
      "pageSize": 10
    }
  }
  ```

## 5. 组件设计

## 7. 权限控制设计

### 7.1 权限检查流程

1. **用户进入视频播放页**
2. **检查用户是否登录**：
   - 未登录：跳转到登录页
   - 已登录：继续下一步
3. **检查课程类型**：
   - 免费课程：直接播放
   - 付费课程：继续下一步
   - 会员课程：继续下一步
4. **检查用户权限**：
   - 付费课程：检查用户是否购买
     - 已购买：直接播放
     - 未购买：提示用户购买
   - 会员课程：检查用户是否为会员
     - 是会员：直接播放
     - 非会员：提示用户开通会员

### 7.2 权限检查实现

#### 7.2.1 客户端权限检查
```javascript
// 权限检查函数
const checkPermission = async (courseId) => {
  // 检查用户是否登录
  const isLoggedIn = useAuthStore.getState().isLoggedIn;
  if (!isLoggedIn) {
    Taro.navigateTo({ url: '/pages/profile/profile' });
    return false;
  }
  
  // 检查课程权限
  try {
    const result = await get(`/api/courses/${courseId}/check-permission`);
    if (result.hasPermission) {
      return true;
    } else {
      // 权限不足，提示用户
      Taro.showModal({
        title: '提示',
        content: result.reason || '您没有权限观看该课程',
        success: (res) => {
          if (res.confirm) {
            // 跳转到课程详情页
            Taro.navigateTo({ url: `/pages/course/detail?id=${courseId}` });
          }
        }
      });
      return false;
    }
  } catch (error) {
    console.error('检查权限失败:', error);
    Taro.showToast({
      title: '检查权限失败',
      icon: 'none'
    });
    return false;
  }
};
```

#### 7.2.2 服务端权限检查
服务端在处理视频播放地址请求时，会再次检查用户权限，确保安全性。

## 9. 性能优化设计

### 9.1 视频加载优化
- **使用CDN加速**：将视频文件存储在CDN上，提高加载速度
- **视频分段加载**：使用HLS加密（m3u8格式），支持分段加载
- **自适应码率**：根据网络情况自动调整视频质量
- **预加载**：在用户点击视频前，预加载视频封面和部分视频数据

### 9.2 页面加载优化
- **懒加载**：图片和视频使用懒加载，减少初始加载时间
- **分页加载**：课程列表和观看历史使用分页加载，减少单次请求的数据量
- **缓存机制**：对课程列表、课程详情等数据进行缓存，减少网络请求
- **组件按需加载**：使用Taro的按需加载功能，减少初始包大小

### 9.3 渲染优化
- **虚拟列表**：对于长列表（如课程列表、观看历史），使用虚拟列表，减少DOM节点数量
- **避免不必要的重渲染**：使用React.memo、useMemo、useCallback等优化组件渲染
- **减少组件层级**：简化组件结构，减少组件嵌套层级
