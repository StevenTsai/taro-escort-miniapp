# 患者问诊病历报告系统前端详细设计文档

## 1. 设计概览

### 1.1 设计目标
本设计旨在实现一个简洁高效的患者问诊病历报告系统前端，支持从患者详情页触发报告生成，预览报告内容，并导出为PDF格式。系统采用Taro框架开发，确保在微信小程序环境中良好运行。

### 1.2 设计原则
- **简洁易用**：减少不必要的操作步骤，提供直观的用户界面
- **响应式设计**：适配不同屏幕尺寸的设备
- **性能优化**：确保在小程序环境中流畅运行
- **一致性**：与现有项目的设计风格保持一致

## 2. 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Taro | 4.1.5 | 前端框架，支持多端适配 |
| Taro UI | 3.1.0 | UI组件库 |
| React | 18.0.0+ | 前端开发库 |
| SCSS | 7.0.0+ | CSS预处理器 |
| Axios | 0.27.0+ | 网络请求库 |

## 3. 页面结构

### 3.1 页面列表
| 页面名称 | 路径 | 主要功能 |
|---------|------|----------|
| 患者详情页 | `/src/pages/patientDetail/patientDetail.jsx` | 展示患者详情，添加报告生成入口 |
| 报告生成页 | `/src/pages/reportGenerate/reportGenerate.jsx` | 展示报告预览，提供PDF导出功能 |

### 4.2 报告生成页组件

#### 4.2.1 页面结构
- **文件路径**：`/src/pages/reportGenerate/reportGenerate.jsx`
- **组件结构**：
  - 页面头部：标题"病历报告生成"
  - 加载状态：数据加载过程中显示
  - 报告预览区域：展示生成的报告内容
  - 操作按钮区域：包含"导出PDF"按钮

#### 4.2.2 状态管理
| 状态名称 | 类型 | 初始值 | 描述 |
|---------|------|--------|------|
| patientId | String | "" | 患者ID |
| loading | Boolean | true | 数据加载状态 |
| reportData | Object | null | 报告数据 |
| exportLoading | Boolean | false | PDF导出加载状态 |

#### 4.2.3 核心功能
1. **数据加载**：
   - 从路由参数获取患者ID
   - 调用API获取患者数据和治疗事件
   - 整合数据生成报告预览

2. **报告预览**：
   - 以结构化方式展示报告内容，参考医院病历格式
   - 包含头部信息、患者基本信息、诊断过程、治疗方案、检查结果等
   - 支持滚动查看完整报告
   - 模拟PDF格式的页面布局

3. **PDF导出**：
   - 调用PDF导出API
   - 显示导出进度
   - 完成后触发文件下载

#### 4.2.4 代码示例
```jsx
import { Button, ScrollView, Text, View, ActivityIndicator } from '@tarojs/components';
import Taro, { useRouter } from '@tarojs/taro';
import { useEffect, useState } from 'react';
import { get, post } from '@/utils/request';
import './reportGenerate.scss';

const ReportGenerate = () => {
  const router = useRouter();
  const patientId = router.params.id;
  const [loading, setLoading] = useState(true);
  const [reportData, setReportData] = useState(null);
  const [downloadLoading, setDownloadLoading] = useState(false);

  useEffect(() => {
    loadReportData();
  }, [patientId]);

  const loadReportData = async () => {
    try {
      setLoading(true);
      // 调用报告生成API
      const response = await post('/api/report/generate', { patient_id: patientId });
      if (response) {
        setReportData(response);
      }
    } catch (error) {
      Taro.showToast({ title: '加载报告数据失败', icon: 'none' });
      console.error('Failed to load report data:', error);
    } finally {
      setLoading(false);
    }
  };

  const handleDownloadPDF = async () => {
    try {
      setDownloadLoading(true);
      // 调用PDF下载API
      const response = await fetch(`/api/report/export-pdf?patientId=${patientId}`);
      if (response.ok) {
        const result = await response.json();
        if (result.downloadUrl) {
          // 触发下载
          window.location.href = result.downloadUrl;
          Taro.showToast({ title: 'PDF下载成功', icon: 'success' });
        }
      } else {
        Taro.showToast({ title: 'PDF下载失败', icon: 'none' });
      }
    } catch (error) {
      Taro.showToast({ title: 'PDF下载失败', icon: 'none' });
      console.error('Failed to download PDF:', error);
    } finally {
      setDownloadLoading(false);
    }
  };



  if (loading) {
    return (
      <View className="loading-container">
        <ActivityIndicator size="large" color="#1890ff" />
        <Text className="loading-text">正在生成报告，请稍候...</Text>
      </View>
    );
  }

  if (!reportData) {
    return (
      <View className="error-container">
        <Text className="error-text">报告数据加载失败</Text>
        <Button onClick={loadReportData}>重新加载</Button>
      </View>
    );
  }

  return (
    <View className="report-generate-container">
      <View className="header">
        <Text className="title">病历报告生成</Text>
      </View>

      <ScrollView className="report-preview" scrollY>
        {/* 报告预览内容 */}
        <View className="report-content">
          {/* 头部信息 */}
          <View className="report-header">
            <View className="header-top">
              <Text className="company-name">壹鹿康行</Text>
              <Text className="company-full-name">佛山市顺德区壹鹿康行健康咨询服务中心</Text>
              <View className="qrcode">
                <Text>二维码</Text>
              </View>
            </View>
            <View className="header-title">
              <Text className="title-text">智能问诊病历报告</Text>
            </View>
          </View>

          {/* 患者基本信息 */}
          <View className="patient-info">
            <Text className="section-title">患者体格</Text>
            <View className="info-row">
              <Text className="info-item">{reportData.patient.name}</Text>
              <Text className="info-item">{reportData.patient.age}岁</Text>
              <Text className="info-item">身高{reportData.patient.height}CM</Text>
              <Text className="info-item">体重{reportData.patient.weight}KG</Text>
            </View>
          </View>

          {/* 诊断及治疗经过 */}
          <View className="diagnosis-treatment">
            {/* 第一阶段：确诊及术前 */}
            <View className="stage-section">
              <Text className="stage-title">第一阶段 确诊及术前</Text>
              {reportData.stages && reportData.stages.first && reportData.stages.first.map((item, index) => (
                <View key={index} className="stage-content">
                  <Text className="date-text">{item.date}</Text>
                  <Text className="content-text">{item.content}</Text>
                </View>
              ))}
            </View>

            {/* 第二阶段：手术 */}
            <View className="stage-section">
              <Text className="stage-title">第二阶段 手术</Text>
              {reportData.stages && reportData.stages.second && reportData.stages.second.map((item, index) => (
                <View key={index} className="stage-content">
                  <Text className="date-text">{item.date}</Text>
                  <Text className="content-text">{item.content}</Text>
                </View>
              ))}
            </View>
          </View>

          {/* 其他基础病及用药 */}
          <View className="other-info">
            <Text className="section-title">其他基础病及用药</Text>
            <Text className="content-text">{reportData.otherDiseases || '未填写'}</Text>
          </View>

          {/* 当前状态 */}
          <View className="current-status">
            <Text className="section-title">当前状态</Text>
            <Text className="content-text">{reportData.currentStatus || '未填写'}</Text>
          </View>

          {/* 医院诊断的疾病名称 */}
          <View className="disease-name">
            <Text className="section-title">医院诊断的疾病名称</Text>
            <Text className="content-text">{reportData.diseaseName || '未填写'}</Text>
          </View>

          {/* 检查报告 */}
          <View className="inspection-reports">
            {reportData.inspections && reportData.inspections.map((inspection, index) => (
              <View key={index} className="inspection-item">
                <Text className="inspection-date">{inspection.date}</Text>
                <Text className="inspection-title">{inspection.title}</Text>
                <Text className="inspection-content">{inspection.content}</Text>
                <Text className="inspection-hospital">{inspection.hospital}</Text>
              </View>
            ))}
          </View>
        </View>
      </ScrollView>

      <View className="footer">
        <Button 
          className="download-btn" 
          onClick={handleDownloadPDF}
          loading={downloadLoading}
          disabled={downloadLoading}
        >
          下载PDF
        </Button>
      </View>
    </View>
  );
};

export default ReportGenerate;
```

## 6. 交互设计

### 6.1 患者详情页交互
1. **报告生成入口**：
   - 在患者详情页的操作按钮区域，点击"生成病历报告"按钮
   - 按钮点击后，跳转到报告生成页，并传递患者ID

### 6.2 报告生成页交互
1. **数据加载**：
   - 页面加载时，显示加载动画
   - 调用API获取报告数据，包括头部信息、患者基本信息、诊断及治疗经过、其他基础病及用药、当前状态、医院诊断的疾病名称、检查报告等
   - 数据加载失败时，显示错误信息和重试按钮

2. **报告预览**：
   - 数据加载成功后，以医院病历格式展示报告内容
   - 头部信息区域显示公司名称、二维码等
   - 患者基本信息区域显示患者姓名、年龄、身高、体重等
   - 诊断及治疗经过区域按阶段展示治疗过程
   - 其他基础病及用药区域显示相关信息
   - 当前状态区域显示患者当前情况
   - 医院诊断的疾病名称区域显示诊断结果
   - 检查报告区域按时间顺序展示检查结果
   - 支持滚动查看完整报告

3. **PDF下载**：
   - 点击"下载PDF"按钮，显示下载加载状态
   - 调用PDF导出API (`/api/report/export-pdf`)
   - 下载成功后，获取返回的`downloadUrl`和`fileName`
   - 根据`downloadUrl`触发文件下载
   - 下载失败时，显示错误提示

4. **页面导航**：
   - 支持通过小程序导航栏返回患者详情页

## 7. API数据格式

### 7.1 报告生成接口
- **接口名称**：`/api/report/generate`
- **请求方法**：`GET`
- **请求参数**：
  | 参数名 | 类型 | 必填 | 说明 |
  |-------|------|------|------|
  | `patientId` | String | 是 | 患者ID |

### 7.2 PDF导出接口
- **接口名称**：`/api/report/export-pdf`
- **请求方法**：`GET`
- **请求参数**：
  | 参数名 | 类型 | 必填 | 说明 |
  |-------|------|------|------|
  | `patientId` | String | 是 | 患者ID |
- **返回类型**：`application/json`
- **响应格式**：
  ```json
  {
    "downloadUrl": "https://example.com/pdf/12345.pdf",
    "fileName": "患者张三_病历报告_2025-07-01.pdf"
  }
  ```
  | 字段名 | 类型 | 说明 |
  |-------|------|------|
  | `downloadUrl` | String | PDF文件下载地址 |
  | `fileName` | String | 下载文件名 |

## 8. 性能优化

### 8.1 前端性能优化
1. **数据加载优化**：
   - 使用异步请求，避免阻塞UI
   - 实现数据缓存，减少重复请求

2. **渲染优化**：
   - 使用React.memo()优化组件渲染
   - 避免不必要的重渲染

3. **图片优化**：
   - 如报告中包含图片，使用适当的压缩和懒加载

4. **网络优化**：
   - 使用HTTP/HTTPS压缩
   - 减少请求数量，合并API调用

### 7.2 小程序特定优化
1. **包体积优化**：
   - 按需引入组件和库
   - 压缩代码和资源

2. **内存优化**：
   - 及时释放不再使用的资源
   - 避免内存泄漏

3. **启动速度优化**：
   - 减少初始化时间
   - 延迟加载非关键资源
